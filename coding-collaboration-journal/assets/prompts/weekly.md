# Weekly Automation Prompt

Operate this installed journal repository in **weekly** mode. Treat the repository as the self-contained runtime: use `.journal/config.toml`, `AGENTS.md`, `docs/method/*`, `scripts/*`, memory, schemas, rubrics, and templates as the authoritative instructions. The original Skill is only needed for installation, upgrades, repair, or workflow evolution.

1. Read the previous completed local week, active memory, patterns, experiments, radar candidates, and report coverage.
2. Consolidate repeated collaboration behaviors with counter-evidence. Do not promote a pattern solely because several daily reports paraphrased the same original event.
3. Compare only similar task classes and note task difficulty, model/client/rubric epochs, and evidence confidence.
4. Review active experiments: observations, sample size, confounders, and whether to continue, revise, close, or reject.
5. Select at most three high-leverage improvements.
6. Run targeted web research only for triggered radar candidates. Prefer current official documentation, primary research, standards, maintainers' engineering material, and reproducible evidence. Save source dates, access dates, applicability, limitations, and citations in practice cards.
7. Update `reviews/weekly/YYYY-Www.md`, bounded memory, pattern state, experiment state, and radar ledger.
8. Validate, commit with `journal: YYYY-Www weekly review`, and push according to policy.

Do not write a generic productivity essay or a single personal performance score.
