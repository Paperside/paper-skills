# Architecture

## Product Definition

The installed system is a personal/team-internal **AI coding collaboration observatory**. Its durable value is the loop below, not the number of reports it produces:

```text
agent sessions + repositories + resources + human notes
                         │
                         ▼
               normalized evidence pack
                         │
             ┌───────────┴───────────┐
             ▼                       ▼
       dated archive            relevant history
             │                       │
             └───────────┬───────────┘
                         ▼
            daily analysis and evaluation
                         │
                         ▼
       patterns → experiments → workflow changes
                         │
                         └─────────────── feedback to future work
```

## Layers

### 1. Discovery Index

Hooks append small JSON events during the day. They make active sessions, timestamps, working directories, transcript locations, prompts, tool calls, and failures discoverable. A hook is not the only source of truth.

### 2. Canonical Readers

- Codex: prefer App Server `thread/list`, `thread/read(includeTurns=true)`, or paged full turns.
- Claude Code: read the local session JSONL files identified by hook events and the configured project store.
- Git: inspect repositories independently of what either agent claimed.
- Resources: recover URLs, documents, MCP calls, commands, test output, and referenced files from the permitted transcripts and tool events.

### 3. Normalizer

Convert provider-specific records into a provider-neutral envelope:

```json
{
  "evidence_id": "E-CODEX-01J...",
  "source": "codex|claude|git|resource|human",
  "source_ref": {"session_id": "...", "turn_id": "..."},
  "occurred_at": "2026-06-20T14:42:18+08:00",
  "project": {"repo_id": "repo-api", "cwd": "/work/api"},
  "kind": "user_prompt|agent_message|tool_call|commit|test|resource",
  "content": {},
  "redaction": {"level": "Low", "changes": []},
  "content_hash": "sha256:..."
}
```

Stable IDs and hashes allow a report to be regenerated without silently changing its evidence trail.

### 4. Evidence Pack

For each report date, store:

```text
journal/YYYY/MM/YYYY-MM-DD/
├── report.md
├── evidence.json
└── run.json
```

At Low privacy, `evidence.json` may contain full permitted messages and tool context. Large raw artifacts can be split under `raw/` while the index retains hashes and paths.

### 5. Analysis

The analyzer receives:

- the report day's evidence;
- relevant prior days, selected by project/task/session links;
- active patterns and experiments;
- the rubric and version metadata;
- selected external-practice evidence when triggered.

It must write facts first, then interpretations, then evaluation and proposed experiments.

### 6. Bounded Memory

The archive can grow indefinitely. Operational memory must stay compact:

- `memory/user-profile.md`: stable preferences and working style;
- `memory/candidates.yaml`: proposed memory changes awaiting promotion, merge, rejection, or staleness handling;
- `memory/project-index.yaml`: active projects, aliases, and continuity pointers;
- `memory/collaboration-patterns.yaml`: supported hypotheses with counter-evidence;
- `memory/experiments.yaml`: active and completed workflow experiments;
- `memory/open-loops.yaml`: unfinished work and recovery context.
- `memory/session-briefing.md`: a precomputed L3 briefing regenerated by the daily run.

This follows the useful distinction between an unlimited searchable history and a small set of curated facts loaded into future runs. `SessionStart` injection is Beta and disabled by default. When explicitly enabled, the hook injects only the precomputed, sanitized `memory/session-briefing.md`; it never builds a briefing from L2 memory on demand. Detailed claims still require retrieval from dated evidence.

### 7. Publisher

The publisher validates artifacts, runs a secret scan, commits deterministic changes, and pushes. It records failure state under `.journal/state/` and retries later.

## Invariants

1. A report can cite only evidence that exists.
2. `no-activity` means all configured collectors completed successfully and found no qualifying activity.
3. Every run records source coverage and software/model/rubric versions.
4. Installed hooks are additive and idempotent.
5. The selected privacy level controls persisted content, not just the final prose.
6. Workflow mutation is separate from observation: analysis may propose changes, but applies them only under the configured autonomy policy.
